alt+me

Diseño y Desarrollo webExploración de alter-egos digitales

Proyecto Final 💀 Deadline 03/06/2025

Explora dentro de tu interior para crear una expresión digital de un alter-ego. El proyecto combinará narrativa personal, diseño web y expresión creativa mediante tecnologías front-end modernas.

Requisitos Técnicos

1. Estructura y Semántica

  • Usar HTML semántico con etiquetas adecuadas (<header>, <main>, <section>, <footer>, etc.).
  • Utilizar atributos obligatorios como title en enlaces y alt en imágenes para mejorar la accesibilidad.
  • Jerarquía adecuada de headings (h1-h6)
  • Optimizar la jerarquía de títulos.
  • Jerarquización de Información
  • Proporcionar descripciones alternativas en imágenes mediante el atributo alt.
  • Utilizar enlaces descriptivos en lugar de textos genéricos como "haz clic aquí".

1. Metadata

    • Incluye un título de página.
    • Incluye definición de lenguaje.
    • Incluir un favicon compatible con navegadores modernos y un fallback para versiones antiguas. Puedes ayudarte de generadores como realfavicongenerator.net
    • Agregar metadata Open Graph para mejorar la previsualización en redes sociales.

2. Estilos

  • Implementación con Tailwind CSS
  • Mínimo 3 componentes/utilities personalizados
  • Paleta de colores coherente con buen contraste de lectura para mejorar la accesibilidad.
  • Favicons personalizados
  • Selecciona una tipografía legible y con estilo coherente.

3. Responsividad

  • Diseño mobile-first
  • Breakpoints para tablet/desktop
  • Imágenes responsive (srcset o <picture>)

4. Contenido Mínimo

  • Página de Inicio (presentación)
  • Sección "Acerca de/About" (explicación conceptual)
  • Página de detalle de Proyecto/Obra/Pieza/Post
  • Página de Contacto (Redes sociales)

5. Funcionalidades

  • Al menos un elemento interactivo
  • Navegación entre páginas funcional
  • Efectos hover/transiciones CSS

6. Despliegue

  • Repositorio GitHub organizado
  • Deploy en GitHub Pages
  • README.md completo

Criterios de Evaluación

  • Funcionalidad general y experiencia de usuario.
  • Calidad del código: estructura, indentación y nomenclatura.
  • Organización del proyecto en carpetas con módulos y clases bien definidos.
  • Diseño, responsividad y accesibilidad:
    • Uso adecuado del tamaño de tipografía.
    • Tamaño recomendado para tamaño de botones en múltiples dispositivos.
    • Contraste suficiente entre colores.
      Verificador de contraste
  • Presencia de metadata y uso de etiquetas semánticas.
  • Originalidad, complejidad y funcionalidad del código entregado.
  • Repositorio y commits en github.
  • Publicación en github pages.
